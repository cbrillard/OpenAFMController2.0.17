Access to Gwyscan refine path edges.vi

creates a new refined scan path based on the difference of two previous scanning paths, representing coarse and fine scans.

Teh function refines the scan path on the basis of edges dedected from previous data. It creates a preview regular array (xreal, yreal, xoffset, yoffset) from the previous scanned data set provided and it searches for edges. It tehn creates a new path crossing the edges with pixel spacing of xstep and ystep

error in can accept error information wired from VIs previously called. Use this information to decide if any functionality should be bypassed in the event of errors from other VIs.

Right-click the error in control on the front panel and select Explain Error or Explain Warning from the shortcut menu for more information about the error.

Input parameters for Gywscan refine path edges

    		xypos

    new path positions

    		xypos_prev

      previous xy positions, stored as interleaved arrays(x1,y1, x2,y2..xn,yn) of size 2n

    		channel_prev

    is previous channel to be used ofr thresholding

    		n_prev

    is the previous number of measured data points or data points to use within a preview function

    		threshold

    is the threshold for the refinement algorithm

    		xstep

    is the desired data step in the x direction in metres

    		ystep

    is the desired data step in the y direction in metres

    		xreal

    is the range in metres

    		yreal

    is the y range in metres

    		xoffset

    is the x offset of the (0,0) pixel in metres

    		yoffset

    is the y offset of the (0,0) pixel in metres

    		zdata_prev

    is previous measured values (height, error signal etc) stored sequentially ( eg za1, za2...zab, zb1, zb2....zbn for channels za and zb)
