Access to Gwyscan create path space filling progressive.vi

creates scan path for equidistant measurement on a space filling curve with progressive resoultions

Teh Hilbert space filling curve is specified in documentation of access to gwyscan create path space filling

Both access to gwyscan gwyscan create path space filling and this function construct a set of equivalent sets of positions. This more complex function permits progressive refinement while keeping the basic properties of the space filling approach.

The finest grid is given by order in the same manner as in access to gwyscan create path space filling

parameter coarse order defines the coarse resolution at which the scanning shoudl start. the path fills teh grid at this resolution completely frist with the Hilbert curve , then it starts adding more positions by progressive refinement. Pass 0 to start from the single point curve and always progress by refinement. Pass the sam value as order to avoid progressive refinement and scan immediately as the finest grid (as access to gwyscan creat path space filling does). The value of coarse order shoudl be always at most equal to order ebcause you cannot start scanning at finer resoultion that the final scan resolution.

Furthermore, it can be specified that you have already measured positions up to order and do not want to measure these positions again. Pass -1 if you want to measure all positions. Generally skip order should be smaller than coarse order because it makes no sense to specify that you want to measure soem points and simultaneously skip them. If you do so skip order takes precedence (and in the extreme case this function can even nstruct a completely empty path)
